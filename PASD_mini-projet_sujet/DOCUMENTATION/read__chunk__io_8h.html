chunk read_chunk_io ( FILE *  f)

Read a token from a stream.

The first (non-space) character is read and depending on it, only some chunk are possible. More characters are read until:

  • chunk is fully read, and
  • there is no ambiguity ("!" from "!=", minus operation from sign of number…). If one (or more) char that could belong to another chunk are read to settle ambiguity, they are put back (with ungetc).

Label (protected or not) are distinguished from keywords by using the table in operator_creator_list.

Static functions are used to handle separately numbers (int or float), label or keyword, sstring, block…

Following spaces are discarded.

If the OEF is reached before anything is read, then a value_error with VALUE_ERROR_IO_EOF is returned.

If the input cannot form a legal chunk, then a corresponding value_error is returned.

Parameters
fstream to read from
Precondition
f is not NULL (assert-ed)
Returns
chunk read (should be a value or an operator) or a value_error if reading fail.
