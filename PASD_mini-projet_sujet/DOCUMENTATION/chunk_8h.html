Detailed Description

Provide the basic layer and mechanism for object oriented programming with chunk as bottom class.

The principle is to respond to a message with a basic_type.

The function chunk_answer_message() is used to trigger the appropriate function.

The term method denotes registered ( message , action ) in any "implementation" of chunk

All chunk's must implement the following methods:

  • "destroy" to destroy the chunk and release ressources,
  • "print" to print the chunk on a stream (FILE *), and
  • "copy" to provide a copy of the chunk (can provide an independent copy or can use a reference counter or…). Destroying an instance or a copy does not destroy the other. This also implies that both should be destroyed before the programs finished.

In each module, a function should be provided to test the nature of the chunk. Its name is of the form:

  • chunk_is_XXX
  • value_is_XXX for sub-classes of value
  • operator_is_XXX for sub-classes of operator
  • …
